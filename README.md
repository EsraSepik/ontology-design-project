# Smart Recipe Ontology (SRO)

## 📌 Project Overview
[cite_start]The purpose of the Smart Recipe Ontology (SRO) is to provide a knowledge model for cooking recipes[cite: 116]. [cite_start]This ontology holds all the necessary knowledge required for cooking a dish, allowing users to query recipes based on various criteria such as diet restrictions and calorie counts[cite: 116].

## 🎯 Scope
[cite_start]The focus of this ontology is the recipe and its structure[cite: 116]. [cite_start]It includes references to required ingredients and utilities, designed specifically for individuals who want to filter recipes based on specific dietary needs[cite: 116].

## 📂 Repository Structure
* `sro.ttl`: The conceptual schema (TBox) defining the core classes and properties.
* `abox.ttl`: The instance-level data (ABox) containing example recipes and ingredients.
* `orsd.pdf`: The Ontology Requirements Specification Document detailing competency questions and rules.
* `sparql_queries/`: Directory containing SPARQL queries to extract information from the knowledge graph.

## 🛠️ Implementation
[cite_start]This project is developed using **Ontology Web Language (OWL)** [cite: 116] and designed via Protégé.
